# 集成游戏主界面任务拆解

> **当前归类**：进行中任务文档（待实现）  
> **使用边界**：勾选状态仅用于跟踪本次任务推进，不替代最终验收结果。  
> **现行真源**：实际代码、`src/router.tsx`

## 任务列表

- [ ] 任务 1：实现游戏主界面页面（GameMain）
  - [ ] 子任务 1.1：确定主界面布局（背景 / HUD / 功能入口区）
  - [ ] 子任务 1.2：接入主城背景资源或复用主城现有组件能力
  - [ ] 子任务 1.3：实现功能入口组件（按钮/九宫格），支持路由跳转

- [ ] 任务 2：重构路由以主界面为默认落点
  - [ ] 子任务 2.1：将 `/` index 指向 GameMain
  - [ ] 子任务 2.2：迁移英雄列表到显式路由（例如 `/heroes`）
  - [ ] 子任务 2.3：新增联盟路由（例如 `/alliance`）并接入 AllianceDashboard
  - [ ] 子任务 2.4：保持现有玩法路由不变（避免破坏深链接）

- [ ] 任务 3：调整应用壳层从“开发导航”向“游戏壳层”迁移
  - [ ] 子任务 3.1：选择并落地 Layout 调整策略（方案 A 或方案 B）
  - [ ] 子任务 3.2：确保钱包连接仍可在主界面与功能页使用
  - [ ] 子任务 3.3：确认“退出回主界面”落点一致（不引入重复初始化）

- [ ] 任务 4：验证与回归
  - [ ] 子任务 4.1：手动验证主界面入口跳转与返回路径
  - [ ] 子任务 4.2：运行构建与测试（build / test / lint 按需）
  - [ ] 子任务 4.3：补充必要的端到端测试用例或更新现有用例（如存在）

## 依赖关系

- 任务 2 依赖任务 1（主界面页面存在后再切换默认落点）
- 任务 3 与任务 2 有耦合（Layout 形态会影响导航入口呈现）
- 任务 4 依赖任务 1-3（实现完成后进行验证）
