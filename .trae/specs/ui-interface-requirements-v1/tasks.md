# 《指尖无双》手游 · 界面整合任务拆解（V1.0）

> 说明：本任务列表用于把 spec 的目标拆解为可执行的实现步骤。条目按“先基础体系，后页面改造，再补通用弹窗/红点与动效”排序。

## 任务清单

### T1. 建立游戏视觉 Token 与验收页

- 在全局样式中新增 `--game-*` 变量（颜色/字体/圆角/间距/按钮最小尺寸/安全区）。
- 更新 Style Guide：展示游戏 token（含品质色、按钮语义色、标题/正文/辅助字档位）。

### T2. 实现 GameShell（TopBar + ResourceBar + BottomBar）

- TopBar：返回/主页、标题、右侧快捷（设置/邮件/福利/头像/充值占位）。
- ResourceBar：金币/元宝/粮食/木材/矿石/包子展示与点击跳转（短期可占位）。
- BottomBar：背包/邮件/好友/福利/活动入口（含红点/角标与锁定态）。
- 安全区适配：Top/Bottom padding 处理；竖屏优先。

### T3. 主界面（GameMain）对齐“12 宫格 + 红点/锁定 + 动效”

- 固化 12 个入口顺序与 2×6 布局，入口卡片最小点击区域 88×88。
- 接入统一入口配置（标题/图标/路由/解锁条件/红点 key）。
- 点击动效与路由淡入切换动效。

### T4. 红点/角标系统（全局）

- 定义统一的红点状态模型与更新来源（任务可领取、邮件新消息、背包可开启等）。
- 入口（主界面/底栏/按钮）统一读取同一来源，避免分散判断。

### T5. 统一弹窗体系（Modal System）

- 建立 ModalHost 与标准 Modal（标题区/内容区/按钮区/关闭行为/动画）。
- 实现 6 类通用弹窗：奖励获得、资源不足、二次确认、奖励预览、战报、竞拍结果。
- 替换现有 alert 与零散 overlay（分阶段迁移：先资源不足/二次确认/奖励获得）。

### T6. 武将界面（Heroes）重构为“左列表 + 右详情”

- 列表：阵营/品质/职业筛选 + 名称搜索；卡片信息对齐需求字段。
- 详情：基础信息/属性（等宽数字）/技能（主动 CD/被动/连招）/养成按钮区。
- 养成：升级消耗包子、升星消耗碎片、每 10 级进阶；成功反馈动效；失败走资源不足弹窗。

### T7. 主城界面（Main City）结构对齐

- 建筑卡片字段对齐：等级、上限、产出、升级判断与不足提示。
- 右侧面板：整合竞拍展示（物品/记录/出价/倒计时/归属），数据可先 mock。

### T8. 联盟界面（Alliance）结构对齐与地图占位

- 顶部信息区字段对齐（含盟主、人数、战力总和）。
- 功能入口区对齐（成员/公告/科技/商店/攻城/退出解散）。
- 世界城池地图：实现可滚动/缩放的地图占位 + 城池点位/列表 + 点击弹窗（宣战/协防/战报）。

### T9. 战斗/采集/攻城 HUD 外层与结算弹窗

- 战斗 HUD：顶部双方信息、底部技能/自动/加速（先 UI，后接入逻辑）。
- 采集：队伍配置（≤5）、进度条、离线收益结算弹窗（可 mock 产出）。
- 攻城：宣战倒计时/集结/战报区域（与联盟地图交互接口预留）。

### T10. 文案集中化与多语言接入点

- 抽离页面标题、入口名、按钮文案为可配置字典。
- 建立最小 `LocaleProvider`（或等价方案）用于后续替换为完整 i18n。

### T11. 验收与回归（含自动化）

- 基础：主界面入口跳转/返回、弹窗关闭、红点显示/消失路径。
- E2E：补齐主界面→模块→退出→回主界面路径（复用现有 cypress 结构）。
- 构建：确保 build 通过。
