## A. 目录与命名规范

- [ ] `public/game-assets/` 目录存在且层级清晰（按 feature/type 分层）。
- [ ] 新增素材文件名均为小写下划线，无空格与中文名。
- [ ] 素材 key 与文件语义一致，无歧义命名。
- [ ] 未在业务代码中新增硬编码静态素材路径。

## B. 清单与类型约束

- [ ] 存在统一素材清单导出入口。
- [ ] 清单字段覆盖 `key/url/type/feature/optional`。
- [ ] 清单中不存在重复 key。
- [ ] 现有背景图条目已迁移到统一清单结构。

## C. 运行时加载与回退

- [ ] `PreloadScene` 已按清单进行素材加载。
- [ ] 必需素材加载失败时有明确错误反馈。
- [ ] 可选素材加载失败时可安全回退，不阻塞场景进入。
- [ ] 在缺失素材情况下仍可进入默认场景并可操作。

## D. 自动化校验与工作流

- [ ] 可通过 `npm run assets:check` 执行素材专项检查。

- [ ] 校验失败时命令返回非零退出码。
