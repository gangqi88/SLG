# Web3 SLG 项目管理技能

> 本技能定义Web3 SLG游戏项目的项目管理规范、流程和最佳实践。

## 技能概述

- **职责**: 项目管理 - 任务规划、进度跟踪、风险管理
- **适用场景**: Web3 SLG游戏开发全周期管理
- **项目类型**: React 19 + TypeScript 5.7 + Phaser 3.90 + Web3 (UniSat/Fractal Bitcoin)

## 核心职责

### 1. 任务规划

#### 开发阶段划分

- **Phase 1: 基础架构** - 项目初始化、技术选型、核心框架搭建
- **Phase 2: 核心系统** - 英雄系统、战斗系统、资源系统、建筑系统
- **Phase 3: 多人交互** - 联盟系统、社交系统、赛季系统
- **Phase 4: Web3集成** - 钱包连接、 NFT集成、代币经济

#### 任务拆分原则

- 每个任务控制在2-4小时完成
- 任务必须有明确的验收标准
- 关联任务需标注依赖关系

#### 优先级定义

| 优先级 | 说明       | 示例                       |
| ------ | ---------- | -------------------------- |
| P0     | 阻塞性问题 | 核心功能无法运行、安全漏洞 |
| P1     | 必须完成   | 重要功能开发、关键bug修复  |
| P2     | 应该完成   | 优化改进、非关键功能       |
| P3     | 可以延后   | 体验优化、未来功能         |

### 2. 进度跟踪

#### 日常管理

- 每日站会同步进度
- 阻塞问题24小时内解决
- 任务状态实时更新

#### 里程碑管理

- 每两周一个交付周期
- 阶段性功能演示
- 版本发布前必须通过代码检查

#### 进度指标

- **代码质量**: ESLint 0 errors, TypeScript 0 errors
- **功能完成度**: 任务完成率 > 90%
- **Bug修复**: 发布前P0/P1必须清零

### 3. 风险管理

#### 风险识别

| 风险类型 | 识别方法             | 应对策略         |
| -------- | -------------------- | ---------------- |
| 技术风险 | 技术调研、概念验证   | 预留技术攻关时间 |
| 进度风险 | 任务燃尽图           | 及时调整资源分配 |
| 依赖风险 | 外部依赖评估         | 关键路径冗余设计 |
| Web3风险 | 钱包兼容性、链上延迟 | 降级方案准备     |

#### 风险监控

- 每周风险评审会议
- 高风险项每日跟踪
- 建立风险预警机制

### 4. Web3项目特有管理

#### 钱包集成风险

- 多钱包兼容性问题
- 链上交易确认延迟
- 网络拥堵影响体验

#### NFT资产安全

- 智能合约审计
- 资产跨链安全
- 用户资产保护机制

#### 经济模型平衡

- 代币产出/消耗循环
- NFT定价策略
- 玩家收益保障

## 项目管理规范

### 代码质量管控

项目采用自动化流程确保代码质量，详细规范见 `.trae/skills/automation-workflow/SKILL.md`。

代码质量标准：

- ESLint: 0 errors (warnings < 15)
- TypeScript: 0 errors
- 单元测试覆盖率 > 70% (如有测试框架)

### 版本管理

采用Git Flow分支模型，详细规范见 `.trae/skills/automation-workflow/SKILL.md`。

分支类型：

- `main`: 生产分支
- `develop`: 开发分支
- `feature/*`: 功能分支
- `fix/*`: 修复分支
- `hotfix/*`: 紧急修复

### 代码审查

所有代码变更必须通过Pull Request审查合并，详细流程见 `.trae/skills/automation-workflow/SKILL.md`。

## 沟通协作

### 同步方式

- 即时沟通: 群组/私聊
- 文档协作: 项目文档/注释
- 代码审查: Pull Request

### 决策流程

1. 问题识别 → 2. 方案调研 → 3. 团队评审 → 4. 决策执行

## 绩效指标

| 指标          | 目标     | 衡量方式     |
| ------------- | -------- | ------------ |
| 任务完成率    | > 90%    | 任务系统统计 |
| Bug修复及时率 | > 95%    | 问题跟踪系统 |
| 代码质量      | 0 errors | CI/CD检查    |
| 发布准点率    | > 90%    | 版本发布记录 |
