# 任务调度器技能

> 本技能定义Web3 SLG项目中任务的智能分配机制，根据任务类型自动路由到合适的技能处理。

## 技能概述

- **职责**: 任务调度 - 任务分类、技能路由、协作编排
- **适用场景**: AI多技能协作场景下的任务自动分配
- **技术栈**: React 19 + TypeScript 5.7 + Phaser 3.90 + Web3

## 任务路由规则

### 单一任务路由

根据任务类型直接路由到对应技能：

```
任务输入 → 关键字识别 → 技能匹配 → 任务分配
```

#### 开发任务路由

任务特征：涉及代码编写、功能实现、Bug修复

**路由到**: `fullstack-code-standards/SKILL.md`

执行流程：
1. 分析需求和技术方案
2. 编写符合规范的代码
3. 运行代码检查
4. 提交代码审查

#### 项目管理任务

任务特征：涉及规划、跟踪、决策

**路由到**: `web3-slg-project-manager/SKILL.md`

执行流程：
1. 评估任务复杂度和优先级
2. 制定任务拆分方案
3. 分配资源和时间
4. 跟踪进度和风险

#### 质量保障任务

任务特征：涉及测试、审查、验证

**路由到**: `game-tester/SKILL.md`

执行流程：
1. 设计测试用例
2. 执行测试
3. 记录和跟踪缺陷
4. 验证修复

#### 基础设施任务

任务特征：涉及CI/CD、工具、自动化

**路由到**: `automation-workflow/SKILL.md`

执行流程：
1. 分析需求
2. 设计工作流
3. 编写脚本和配置
4. 测试和部署

#### 文档任务

任务特征：涉及文档编写、审查、更新

**路由到**: `document-reviewer/SKILL.md`

执行流程：
1. 分析文档需求
2. 编写或更新文档
3. 审查质量
4. 发布和维护

任务涉及多个领域时，触发多技能协作：

| 任务描述 | 识别结果 | 路由策略 |
|----------|----------|----------|
| 实现英雄升级功能 | 开发任务 | 代码规范 |

| 修复战斗计算Bug | 开发任务+质量保障 | 代码规范 → 测试规范 |

## 多技能协作流程

### 协作模式

#### 串行协作

A技能完成 → B技能继续

适用于：有明确先后顺序的任务

```
示例：编写功能 → 编写测试
代码规范(开发) → 测试规范(验证)
```

#### 并行协作

A技能 + B技能 同时处理

适用于：相互独立的任务

```
示例：同时开发多个功能
代码规范(功能A) ↔ 代码规范(功能B)
```

#### 协作链

A技能 → B技能 → C技能

适用于：复杂的多阶段任务

```
示例：新功能开发流程
代码规范(开发) → 测试规范(测试) → 自动化工作流(部署)
```

### 协作示例

#### 示例1：开发新英雄系统

```
任务：实现完整的英雄系统

阶段1：需求分析
→ 项目管理：规划任务和里程碑

阶段2：功能开发  
→ 代码规范：编写英雄相关代码
  - Hero类型定义
  - HeroDataManager实现
  - 英雄界面组件

阶段3：质量保障
→ 测试规范：编写单元测试和集成测试
→ 文档审查：审查代码质量

阶段4：部署
→ 自动化工作流：更新CI/CD配置
```

#### 示例2：优化战斗性能

```
任务：优化战斗系统性能

阶段1：问题分析
→ 测试规范：性能测试和分析

阶段2：优化实现
→ 代码规范：性能优化代码

阶段3：验证
→ 测试规范：性能验证测试

阶段4：发布
→ 自动化工作流：部署更新
```

## 任务分配执行

### 执行步骤

#### 1. 任务解析

```
输入：任务描述
输出：任务类型、技能需求、优先级
```

解析方法：
- 关键字匹配
- 语义分析
- 上下文推断

#### 2. 技能匹配

```
输入：任务类型、技能需求
输出：最佳技能选择
```

匹配策略：
- 精确匹配：任务特征明确
- 模糊匹配：任务特征不明确
- 多重匹配：需要多技能协作

#### 3. 任务分配

```
输入：技能选择、任务特征
输出：分配方案、执行顺序
```

分配原则：
- 单一职责：每个任务分配给最擅长的技能
- 负载均衡：避免单技能过载
- 效率优先：减少技能切换开销

### 任务状态管理

| 状态 | 说明 |
|------|------|
| pending | 待分配 |
| running | 执行中 |
| waiting | 等待协作 |
| completed | 已完成 |
| failed | 失败 |

## 智能分配决策

### 决策矩阵

| 场景 | 决策 |
|------|------|
| 任务明确属于单一技能 | 直接分配给对应技能 |
| 任务跨多个技能 | 建立协作链，串行处理 |
| 任务需要并行处理 | 同时分配给多个技能 |
| 任务需要先分析 | 先分配给项目管理分析 |
| 任务结果需要验证 | 分配给测试规范验证 |

### 优先级规则

1. **紧急任务**：直接分配，优先处理
2. **阻塞任务**：优先解决，分配最擅长技能
3. **依赖任务**：按依赖关系顺序分配
4. **批量任务**：并行分配提高效率

## 技能协作接口

### 任务传递格式

```typescript
interface Task {
  id: string;
  type: 'development' | 'project' | 'quality' | 'infrastructure' | 'documentation' | 'composite';
  description: string;
  priority: 'P0' | 'P1' | 'P2' | 'P3';
  skills: string[];
  dependencies?: string[];
  status: 'pending' | 'running' | 'waiting' | 'completed' | 'failed';
  result?: any;
}

interface TaskResult {
  taskId: string;
  skill: string;
  success: boolean;
  output: any;
  nextTasks?: string[];
}
```

### 协作协议

- 任务完成后输出标准结果格式
- 后续任务从结果中获取输入
- 失败任务触发重试或升级流程
