# R5 审查流程与常见问题

## 审查流程

1. 作者完成文档修改
2. 对照 `SKILL.md` 索引判断应读取的规则分片
3. 执行格式与版本检查
4. 提交 `document-reviewer` 审查
5. 审查通过后发布

## 审查清单

- 语言是否符合 `R1-language.md`
- 结构是否符合 `R2-format.md`
- 版本尾注是否符合 `R3-version.md`
- 边界与交接是否符合 `R4-collaboration.md`
- 治理门禁是否符合 `R6-change-safety.md`

## 否决条件

- 无灰度放量方案且直接全量发布
- 无关键观测指标或日志验证路径
- 无回滚路径且无前向恢复方案
- 接口或数据结构变更缺少兼容策略

## 常见问题

- **问题**：规则冲突  
  **处理**：按 `SKILL.md` 优先级执行
- **问题**：职责重叠  
  **处理**：保留主技能职责，删除重复段落
- **问题**：文档过长  
  **处理**：主索引保持精简，细节下沉到分片
