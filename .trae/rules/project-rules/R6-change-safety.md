# R6 变更与发布安全规则

## 适用范围

- 适用于所有功能开发、缺陷修复、重构、接口与数据结构变更
- 默认按风险分级执行：低风险 / 中风险 / 高风险

## 1. 变更粒度与正交性

- 单次提交或合并请求只处理一类目标
- 修复问题不得夹带重构或样式清理
- 功能改动不得顺带修改无关模块
- 大功能必须使用特性开关，未完成能力默认关闭
- 优先增量改造，不做无边界重写

## 2. 发布策略与灰度推进

- 生产发布必须采用渐进放量，禁止一次性全量切换
- 上线前必须定义核心观测指标：错误率、延迟、关键业务成功率
- 无可观测项的改动不得上线
- 业务策略改动优先采用 A/B 测试验证收益

## 3. 兼容性与防御性

- 接口与数据结构变更必须同时满足读兼容与写兼容
- 数据库迁移遵循 Add -> Backfill -> Deploy -> Use -> Drop
- 外部输入默认不可信，必须有异常兜底
- 非核心字段解析失败不得导致核心流程中断
- 核心逻辑必须具备单元测试，跨服务调用必须具备契约测试

## 4. 可逆性与恢复

- 任何中高风险改动必须提供可执行回滚路径
- 回滚不可行时必须提供前向恢复与补偿方案
- 回滚或恢复步骤必须在上线前演练或验证

## 5. 协作与审查

- 代码审查必须确认改动范围最小化与影响面评估
- 中高风险改动必须先有技术设计文档
- 禁止依赖“应该没问题”的主观判断
- 发布决策必须由测试与观测证据支撑

## 6. 风险分级触发要求

| 风险级别 | 典型场景 | 必做项 |
| -------- | -------- | ------ |
| 低风险 | 文案、样式、小范围非核心逻辑 | 单元测试 + 基础验证 |
| 中风险 | 核心流程逻辑调整、外部依赖变更 | 特性开关 + 灰度发布 + 回滚方案 |
| 高风险 | 数据结构变更、接口协议调整、跨模块改造 | 设计评审 + 契约测试 + 灰度放量 + 回滚或前向恢复演练 |

## 7. 执行边界

- 本分片定义全局治理规则，不承载具体技能实现细节
- 具体执行动作由各技能文档按职责落地
