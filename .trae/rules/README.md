# PizzaSwap 项目规则文档

本目录包含 PizzaSwap 套利项目的开发规范与最佳实践文档。根据单一职责原则，规则已拆分为多个专门文档，便于查找和维护。

## 📋 规则文档索引

### 1. [编码标准规范](coding-standards.md)
- 语言要求、代码质量、复杂度控制、代码整洁

### 2. [错误处理规范](error-handling.md)
- 错误处理基本原则、标准化错误处理、网络请求规范、业务异常告警、诊断日志增强

### 3. [日志规范](logging-standards.md)
- Logger 实例管理、日志格式、诊断日志增强、日志级别区分

### 4. [架构设计规范](architecture-design.md)
- 消除双轨制原则、架构设计原则、工具服务复用

### 5. [执行状态管理规范](execution-management.md)
- 执行状态区分、日志级别控制、并发控制原则

### 6. [可观测性要求](observability-requirements.md)
- 诊断信息要求、配置管理、链路追踪

### 7. [测试与验证要求](testing-requirements.md)
- 单元测试、集成测试、性能测试

### 8. [监控与告警规范](monitoring-alerting.md)
- 业务指标监控、依赖健康监控、告警分级

## 📖 使用建议

1. **开发新功能时**：参考《编码标准规范》和《架构设计规范》
2. **编写业务逻辑时**：参考《错误处理规范》和《日志规范》
3. **实现执行链路时**：参考《执行状态管理规范》和《可观测性要求》
4. **质量保障时**：参考《测试与验证要求》和《监控与告警规范》
