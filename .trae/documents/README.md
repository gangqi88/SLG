# `.trae/documents` 目录说明

本目录用于存放**规划、专题分析、阶段性方案、历史过程材料**，默认不作为当前实现真源。

## 目标读者

- 需要回溯历史方案的开发者
- 做专题设计或范围评估的协作者
- 需要了解背景上下文的 AI / 自动化代理

## 使用边界

- 可以用于理解历史决策与候选方案
- 不应用于直接判断“当前代码已经这样实现”
- 若与代码、配置或 `docs/` 冲突，应优先以代码与 `docs/` 为准

## 与其他文档层的关系

- `README.md`：仓库入口
- `docs/`：当前有效说明
- `.trae/rules/`：稳定规则真源
- `.trae/skills/`：任务执行方法
- `.trae/documents/`：规划与专题材料
- `.trae/specs/`：过程性 spec / tasks / checklist

## 当前文件类型

为避免误读，建议在本目录内统一使用以下状态理解：

- **规划方案**：面向未来设计，不代表已落地
- **历史方案**：与现行实现已有偏差，只供回溯参考
- **过程记录**：记录某次修订或变更，不作为标准文档
- **策划背景文档**：用于承载玩法/世界观/数值背景，不替代技术说明

## 维护建议

- 新增文档时，在标题或开头明确标记“规划 / 草案 / 历史 / 过程”属性
- 若某方案已落地，应把结论同步下沉到 `docs/`，而不是只留在本目录
- 若文档已明显失效，可在文件顶部补“已过时”说明，而不是让其继续与现行文档并列混用
