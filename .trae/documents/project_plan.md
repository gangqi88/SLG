# Web3 SLG 游戏项目开发计划

本文档基于 `Web3游戏项目-SLG.md` 制定，旨在构建一个基于 Phaser 3 + React + TypeScript 的 Web3 SLG 游戏。

## 项目概述
- **游戏类型**: SLG (策略游戏)
- **技术栈**: React, Phaser 3, TypeScript, Vite, bitcoinjs-lib (Web3)
- **核心玩法**: 三族对抗 (人族、天使、恶魔)、英雄养成、城池建设、资源争夺
- **Web3特性**: 英雄NFT化、资源上链、UniSat钱包集成

---

## 开发阶段规划

### 第一阶段：基础架构与英雄数据 (Phase 1: Foundation & Hero Data)
**目标**: 完成项目初始化，建立核心数据结构，实现英雄数据的展示。

1.  **项目结构初始化**
    - [ ] 建立目录结构: `src/components`, `src/game/scenes`, `src/types`, `src/data`, `src/utils`.
    - [ ] 配置全局样式与主题 (Tailwind 或 CSS Modules).

2.  **数据结构定义 (TypeScript Interfaces)**
    - [ ] 定义 `Hero` 接口 (属性: 统御, 武力, 谋略, 防御, 种族, 品质等).
    - [ ] 定义 `Skill` 接口 (主动, 被动, 天赋, 冷却, 效果).
    - [ ] 定义 `Bond` (羁绊) 接口.

3.  **英雄数据录入**
    - [ ] 录入 **人族** 10名英雄数据 (苏墨, 秦烈, 温竹, 等).
    - [ ] 录入 **天使** 10名英雄数据 (洛曦, 迦南, 米迦尔, 等).
    - [ ] 录入 **恶魔** 10名英雄数据 (摩洛克, 卡隆, 莉莉丝, 等).
    - [ ] 实现 `HeroService` 用于获取和筛选英雄数据.

4.  **英雄 UI 实现**
    - [ ] 开发 `HeroList` 组件：展示英雄列表，支持按种族/品质筛选.
    - [ ] 开发 `HeroDetail` 组件：展示英雄详细属性、技能、背景故事、立绘(占位).

### 第二阶段：核心战斗逻辑 (Phase 2: Core Combat Logic)
**目标**: 实现基于文档规则的战斗模拟与可视化演示。

1.  **战斗核心逻辑 (纯逻辑层)**
    - [ ] 实现伤害公式: `普攻 = 武力 * 0.8`, `技能 = 谋略 * 系数`.
    - [ ] 实现防御公式: `减免 = 防御 / (防御 + 200)`.
    - [ ] 实现克制关系: 恶魔>人族>天使>恶魔.
    - [ ] 实现回合制/半即时制战斗循环 (包含技能冷却管理).

2.  **战斗场景 (Phaser Scene)**
    - [ ] 创建 `BattleScene`.
    - [ ] 实现单位简单的渲染 (使用占位图形代表不同兵种/英雄).
    - [ ] 实现攻击、技能释放的动画效果 (基础).
    - [ ] 显示战斗飘字 (伤害量, 暴击, 治疗).

3.  **编队系统**
    - [ ] 实现单队配置逻辑: 1主将 + 2副将.
    - [ ] 实现羁绊激活检测逻辑.

### 第三阶段：资源与内政系统 (Phase 3: Resource & Internal Affairs)
**目标**: 实现SLG的经济循环系统。

1.  **资源系统**
    - [ ] 定义资源类型: 木材, 石料, 粮食, 铜币.
    - [ ] 实现资源自动产出逻辑 (每秒/每小时).
    - [ ] 实现资源上限与仓库保护逻辑.

2.  **内政英雄加成**
    - [ ] 实现内政英雄 (如苏墨, 梁石) 的天赋/技能对资源产出的加成效果.
    - [ ] 实现驻扎/委任逻辑.

3.  **主城场景 (City Scene)**
    - [ ] 创建 `CityScene` (Phaser).
    - [ ] 放置基础建筑 (城堡, 兵营, 资源点).
    - [ ] 实现建筑点击交互与升级逻辑 (模拟).

### 第四阶段：Web3 集成 (Phase 4: Web3 Integration)
**目标**: 实现钱包连接与资产映射。

1.  **钱包集成**
    - [ ] 集成 UniSat 钱包连接功能.
    - [ ] 实现签名验证流程.

2.  **NFT 模拟与映射**
    - [ ] 定义英雄 NFT Metadata 标准.
    - [ ] 实现"英雄转NFT"的模拟流程.
    - [ ] 模拟链上资产读取 (在没有测试网环境时使用本地模拟).

### 第五阶段：UI 完善与综合调试 (Phase 5: UI Polish & Polish)
**目标**: 提升用户体验，整合所有模块。

1.  **UI 美化**
    - [ ] 统一 UI 风格 (依据三种族风格设计).
    - [ ] 优化交互体验 (弹窗, 提示, 动画).

2.  **功能整合**
    - [ ] 串联 登录 -> 主城 -> 编队 -> 战斗 完整流程.
    - [ ] 调试数值平衡，确保战斗体验符合预期.

3.  **桌面端 Web3 测试 (Desktop Only)**
    - [ ] 移除所有移动端适配代码 (No Media Queries, No Touch Events).
    - [ ] 确保在 1024x768 - 2560x1440 分辨率下无横向滚动条.
    - [ ] CI/CD 集成桌面分辨率自动化测试.
