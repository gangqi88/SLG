### 0.2 链上操作触发场景

| 操作类型 | 触发场景 | Gas消耗 | 服务费收入 |
|----------|----------|---------|------------|
| 英雄升级 | 对战胜利/任务 | 低 | 低 |
| 英雄进化 | 收集材料 | 中 | 中 |
| 装备强化 | 对战掉落 | 中 | 中 |
| 技能学习 | 获得技能 | 低 | 低 |
| 繁育 | 消耗代币 | 高 | 高 |
| 交易市场 | 买卖NFT | 高 | **最高** |
| 租赁 | 出租英雄 | 中 | 中 |

### 0.2.1 对战驱动收入模型

```
核心公式: 收入 = Σ(用户数 × 对战频率 × 链上操作率 × 单次Gas费)

关键指标:
• DAU (日活) → 基础用户量
• 对战频率 → 玩家活跃度
• 链上操作率 → 每场对战触发的链上操作比例
• 单次Gas费 → FB网络费率

优化方向:
↑ 增加用户数 → 免费用户激励
↑ 提高对战频率 → 匹配算法优化
↑ 链上操作率 → 更多可升级/进化点
↑ 单次Gas费 → 复杂链上操作
```

### 0.2.2 对战场景收入设计

| 对战阶段 | 链上操作 | 触发条件 | 收入等级 |
|----------|----------|----------|----------|
| 开战 | 无 | - | 0 |
| 胜利 | 经验+1 | 首次胜利日 | ★ |
| 升级 | 等级上链 | 升级时 | ★ |
| 进化 | 进化上链 | 进化时 | ★★ |
| 掉落 | 掉落NFT | 稀有掉落 | ★★★ |
| 交易 | 市场挂售 | 卖出时 | ★★★★ |
| 繁育 | 繁育上链 | 繁育时 | ★★★★★ |

#### 战斗胜利链上操作示例:
```
玩家A vs 玩家B (玩家A胜利)

1. 经验+100 → 链上记录 (收入: 低)
2. 升级到Lv.20 → 链上更新 (收入: 低)
3. 掉落蓝色技能NFT → 链上Mint (收入: 中)
4. 玩家装备技能 → 链上绑定 (收入: 低)
5. 玩家挂售旧装备 → 链上交易 (收入: 高)

本场对战总收入: ~5000-10000 sat
```

## 1. 设计原则：用户体验优先

### 1.1 用户体验核心理念

```
┌─────────────────────────────────────────────────────────────┐
│                    用户体验优先原则                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 资产真正归玩家所有                                         │
│     └── 链上可验证、可转移、可交易                              │
│                                                             │
│  2. 透明度与可验证性                                           │
│     └── 所有关键数据链上可查，拒绝暗箱操作                        │
│                                                             │
│  3. 丰富的资产体验                                            │
│     └── 英雄、装备、技能、外观全部NFT化                         │
│                                                             │
│  4. 真正的数字资产                                            │
│     └── 可交易、可组合、可积累价值                              │
│                                                             │
│  5. 跨游戏/跨平台                                             │
│     └── 资产可流通，不被单一游戏绑定                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 传统Web2 vs Web3用户体验对比

| 维度 | 传统Web2 | Web3 (用户体验优先) |
|------|-----------|---------------------|
| 资产所有权 | 开发商 | **玩家真正拥有** |
| 数据透明度 | 服务器黑盒 | **全链上可查** |
| 资产流通 | 不可交易 | **自由市场交易** |
| 账号体系 | 邮箱/手机 | **钱包地址** |
| 资产继承 | 不可转移 | **真正可继承/出售** |
| 成就证明 | 截图 | **链上徽章NFT** |
| 装备绑定 | 绑定角色 | **可拆卸/可交易** |
| 技能获取 | 固定 | **可自由组合** |

## 3. 英雄NFT设计

### 3.1 英雄数据结构

```typescript
// 链上完整数据结构
interface HeroNFT {
  // 基础信息
  id: string;                    // 唯一ID (inscription_id)
  templateId: number;             // 模板ID (1-30)
  name: string;                  // 英雄名称
  
  // 所有权
  owner: string;                  // 钱包地址
  createdAt: number;              // Mint时间
  transferCount: number;          // 转移次数
  
  // 核心属性
  level: number;                  // 等级 (1-80)
  experience: number;            // 经验值
  rarity: number;                 // 稀有度 (1-5: 紫/橙/红/传说/神话)
  stars: number;                 // 星级 (1-5)
  
  // 四维属性 (链上可验证)
  attributes: {
    command: number;              // 统御
    strength: number;             // 武力
    strategy: number;             // 谋略
    defense: number;             // 防御
  };
  
  // 技能绑定 (技能NFT ID)
  skills: {
    active?: string;              // 主动技能NFT ID
    passive1?: string;            // 被动技能1 NFT ID
    passive2?: string;            // 被动技能2 NFT ID
    talent?: string;              // 天赋NFT ID
  };
  
  // 装备绑定 (装备NFT ID)
  equipment: {
    weapon?: string;              // 武器NFT ID
    armor?: string;               // 护甲NFT ID
    accessory?: string;          // 饰品NFT ID
    mount?: string;               // 坐骑NFT ID
  };
  
  // 时装 (时装NFT ID)
  appearance?: string;
  
  // 状态 (链上可见)
  status: {
    loyalty: number;             // 忠诚度 (0-100)
    fatigue: number;             // 疲劳度 (0-100)
    awakenLevel: number;         // 觉醒等级 (0-3)
  };
  
  // 历史数据
  history: {
    totalBattles: number;        // 总战斗场次
    wins: number;                // 胜利场次
    kills: number;               // 总击杀
    highestRank: number;         // 最高段位
    breedingCount: number;       // 繁育次数
  };
  
  // 元数据
  metadata: {
    bloodline?: number;           // 血统ID
    breedParents?: [string, string]; // 父母英雄ID
  };
}
```

```typescript
// 获取英雄数据
async function getHero(heroId: string): Promise<HeroNFT> {
  const inscription = await unistat.getInscription(heroId);
  return parseHeroData(inscription);
}

// 装备技能
async function equipSkill(heroId: string, skillId: string, slot: string): Promise<void> {
  // 验证所有权
  const hero = await getHero(heroId);
  if (hero.owner !== currentWallet) throw new Error('Not owner');
  
  // 验证技能
  const skill = await getSkill(skillId);
  if (skill.owner !== currentWallet) throw new Error('Skill not owned');
  
  // 执行装备
  await unistat.inscribe({
    p: 'slg-hero',
    op: 'equip',
    hero: heroId,
    skill: skillId,
    slot: slot
  });
}

// 交易装备
async function tradeEquipment(equipmentId: string, price: number): Promise<void> {
  const listing = {
    p: 'slg-market',
    op: 'list',
    asset: equipmentId,
    price: price,
    seller: currentWallet,
    royalty: 0.05
  };
  await unistat.inscribe(listing);
}

// 验证数据
async function verifyHeroData(heroId: string): Promise<boolean> {
  const onChainData = await getHero(heroId);
  const localData = getLocalData(heroId);
  return onChainData.hash === calculateHash(localData);
}
```
