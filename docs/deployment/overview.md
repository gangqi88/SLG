# 部署与运行说明

本文档描述当前前端主工程的运行方式、构建产物与部署边界，面向开发、测试与部署维护人员。

## 当前部署模型

主前端工程是一个基于 Vite 构建的静态前端应用：

- 开发时通过 `npm run dev` 启动本地开发服务器
- 生产构建通过 `npm run build` 输出到 `dist/`
- 本地预览通过 `npm run preview`

当前代码库中**未确认存在已落地的持续集成/持续交付流水线**，因此本文件只描述本地可验证的部署事实，不推断未落地的平台流程。

## 运行配置事实

以 `vite.config.ts` 为准：

- 开发端口：`8080`
- 构建目录：`dist/`
- 静态资源目录：`dist/assets/`
- 路径别名：`@ -> ./src`

## 本地构建与预览

### 生产构建

- `npm run build`

构建完成后产物位于 `dist/`。

### 本地预览

- `npm run preview`

用于验证打包后的静态站点能否正常打开、路由能否正常进入、资源能否被正确加载。

## 静态托管要求

当前路由由 `createBrowserRouter` 提供，因此部署到静态托管平台时，应满足以下要求：

1. 站点根目录指向 `dist/`
2. 对非资源请求启用 `index.html` 回退
3. 静态资源目录 `assets/` 可直接被访问

如果托管平台不支持 SPA 回退，直接访问如 `/city`、`/battle` 等路径时可能返回 404。

## 运行时边界

### 浏览器环境

- 主应用面向现代浏览器运行
- 钱包能力依赖桌面浏览器中的 UniSat 扩展环境

### Web3 运行限制

- 当前前端钱包能力基于 `window.unisat`
- 未安装 UniSat 时，钱包连接功能不可用
- 钱包相关能力属于增强能力，不应阻塞静态资源本身加载

### 独立子项目说明

- `test/` 是独立 Hardhat 子项目
- 其环境变量、网络、部署命令与主前端部署流程分开维护
- 主前端发布不等于智能合约子项目已部署

## 部署前最小检查

建议至少执行以下命令：

- `npm run build`
- `npm run preview`

若改动涉及类型、样式或代码质量，再补充：

- `npx tsc --noEmit`
- `npm run lint`
- `npm run lint:style`
- `npx vitest run`
